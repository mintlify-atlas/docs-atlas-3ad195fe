---
title: "Mass password reset"
description: "How to notify all active Signon users that their passwords will be reset and then execute the reset via the Rails console."
---

A mass password reset may be required in response to a security incident or as a proactive security measure. The process has two distinct phases: sending an advance warning email to users, and then performing the actual password reset.

<Warning>
  A mass password reset is a disruptive operation. All affected users will be logged out and must set a new password before they can continue working. Coordinate with your incident response process and communicate the time and date clearly in advance.
</Warning>

---

## Phase 1: Send advance warning emails

### The `PasswordResetEmail` mailer

Use the following mailer class in a Rails console session. The email copy has been written and signed off by the GOV.UK content team for use in mass password reset scenarios.

```ruby
class PasswordResetEmail < ActionMailer::Base
  def notification_email(user)
    to = user.email
    mail(
      to: to,
      from: 'GOV.UK Signon <incident-response@govuk.zendesk.com>',
      reply_to: 'incident-response@govuk.zendesk.com',
      subject: 'Your GOV.UK Signon password will be reset',
      content_type: 'text/plain',
      body: %Q{Hi,

Your Signon password will be reset at TIME on DAY MONTH. You'll be
able to work on GOV.UK as normal until that time. You don't need to
take any action now.

This is happening to all Signon accounts and is part of ongoing work
to make GOV.UK more secure.
On DAY MONTH, you'll be logged out of Signon. You'll then get an email
telling you how to get a new password.

Once you have a new password you'll be able to continue to work on
GOV.UK as normal.

INSERT YOUR SIGNATURE HERE
},
    )
  end
end
```

<Note>
  Replace `TIME`, `DAY MONTH`, and `INSERT YOUR SIGNATURE HERE` with the actual values before sending. Do not alter the rest of the copy — it has been approved for this purpose.
</Note>

### Test on yourself first

Before sending to all users, verify the email renders correctly by sending it to your own account:

```ruby
User.where(email: 'user.name@digital.cabinet-office.gov.uk').each do |user|
  begin
    puts "Attempting to email: #{user.email}"
    PasswordResetEmail.notification_email(user).deliver
  rescue Exception => err
    puts "FAILED trying to email: #{user.email}. Error: #{err}"
  end
end
```

### Send to recently active users

Target users who have signed in within the last 90 days:

```ruby
User.last_signed_in_after(90.days.ago).each do |user|
  begin
    puts "Attempting to email: #{user.email}"
    PasswordResetEmail.notification_email(user).deliver
  rescue Exception => err
    puts "FAILED trying to email: #{user.email}. Error: #{err}"
  end
end
```

<Tip>
  The `rescue Exception` block ensures a delivery failure for one user does not abort the loop. Review the console output afterwards to check for any `FAILED` lines and follow up manually if needed.
</Tip>

## Phase 2: Perform the password reset

At the agreed time, open a Rails console and run the reset for each affected user. The snippet below generates a random password, forces a re-authentication, and sends the standard Devise password reset email:

```ruby
random_password = User.send(:generate_token, 'encrypted_password').slice(0, 20)
user.password = random_password
user.password_confirmation = random_password
if user.save
  ReauthEnforcer.perform_on(user)
  user.send_reset_password_instructions
else
  puts user.errors.full_messages
end
```

To apply this to all recently active users:

```ruby
User.last_signed_in_after(90.days.ago).each do |user|
  random_password = User.send(:generate_token, 'encrypted_password').slice(0, 20)
  user.password = random_password
  user.password_confirmation = random_password
  if user.save
    ReauthEnforcer.perform_on(user)
    user.send_reset_password_instructions
    puts "Reset password for: #{user.email}"
  else
    puts "FAILED for #{user.email}: #{user.errors.full_messages.join(', ')}"
  end
end
```

<Warning>
  Running this snippet immediately logs every affected user out of all active sessions. Only run it at the time communicated in the advance warning email.
</Warning>

## Step-by-step summary

<Steps>
  <Step title="Open a Rails console">
    ```bash
    kubectl exec -it deploy/signon -- rails c
    ```
  </Step>
  <Step title="Define the PasswordResetEmail mailer">
    Paste the mailer class definition from Phase 1 into the console.
  </Step>
  <Step title="Test on yourself">
    Send the notification email to your own address and confirm it arrives and looks correct.
  </Step>
  <Step title="Send the advance warning to all active users">
    Run the `User.last_signed_in_after(90.days.ago)` loop to send the warning email.
  </Step>
  <Step title="Wait until the communicated reset time">
    Do not proceed until the date and time stated in the warning email.
  </Step>
  <Step title="Perform the mass password reset">
    Run the reset loop to generate new random passwords, enforce re-authentication, and trigger individual reset emails for each user.
  </Step>
  <Step title="Monitor for failures">
    Review console output for any `FAILED` lines. Manually reset affected accounts via the Signon UI or individual Rails console commands.
  </Step>
</Steps>
