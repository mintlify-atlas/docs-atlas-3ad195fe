Signon is the identity and access management service that powers GOV.UK publishing. It provides OAuth2-based single sign-on, two-step verification, and fine-grained permission management for all GOV.UK applications.

## How Signon works

Signon sits at the centre of GOV.UK's publishing infrastructure. Applications delegate authentication and authorisation to Signon via OAuth2, and Signon returns user identity and permission data to each application.

<Steps>
  <Step title="Register your application">
    Add your application to Signon using the `applications:create` rake task to receive a client ID and secret.
  </Step>
  <Step title="Configure OAuth credentials">
    Set the `OAUTH_ID` and `OAUTH_SECRET` environment variables in your application using the credentials returned by Signon.
  </Step>
  <Step title="Invite users">
    Create user accounts in Signon and grant them the `signin` permission for your application.
  </Step>
  <Step title="Users authenticate">
    Users visit your application, are redirected to Signon to sign in, and return with a valid session and their permissions.
  </Step>
</Steps>
