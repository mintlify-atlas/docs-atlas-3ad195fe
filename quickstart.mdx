title: "Quick start"
description: "Register an application, obtain OAuth credentials, and get your first user authenticated through Signon."
---

This guide walks you through registering an application with Signon, configuring your application to use the credentials, and granting a user access.

<Note>
All commands must be run inside the [GOV.UK Docker environment](https://github.com/alphagov/govuk-docker). Signon requires access to its database and other services that Docker provides.
</Note>

## Register your application

<Steps>
  <Step title="Create the application record">
    Run the `applications:create` rake task from the Signon repository, substituting your application's details:

    ```bash
    bundle exec rake applications:create \
      name="My App" \
      description="A publishing tool for GOV.UK" \
      home_uri="https://my-app.dev.gov.uk" \
      redirect_uri="https://my-app.dev.gov.uk/auth/gds/callback"
    ```

    The task creates the application record in Signon's database and prints the OAuth credentials:

    ```
    Application 'My App' created.

    config.oauth_id     = 'abc123def456...'
    config.oauth_secret = 'xyz789uvw012...'
    ```

    <Warning>
    The client secret is only shown once. Copy it immediately and store it securely — for example in your application's Kubernetes secrets or a secrets manager.
    </Warning>
  </Step>

  <Step title="Set the OAuth credentials in your application">
    Add the client ID and secret to your application's environment:

    ```bash
    export OAUTH_ID="abc123def456..."
    export OAUTH_SECRET="xyz789uvw012..."
    ```

    In production environments, inject these as secrets rather than plain environment variables. The [gds-sso](https://github.com/alphagov/gds-sso) gem reads `OAUTH_ID` and `OAUTH_SECRET` automatically.
  </Step>

  <Step title="Configure your application for local development">
    If you do not yet have real OAuth tokens configured, run the following script from the Signon repository to seed your local database with development OAuth credentials that match the defaults used by GOV.UK applications in development mode:

    ```bash
    bundle exec ./script/make_oauth_work_in_dev
    ```

    Then start your application with the `GDS_SSO_STRATEGY` environment variable set to `real`:

    ```bash
    GDS_SSO_STRATEGY=real bundle exec rails s
    ```

    <Tip>
    Without `GDS_SSO_STRATEGY=real`, the gds-sso gem uses a stub strategy that bypasses Signon entirely. Set it to `real` to use the full OAuth flow in development.
    </Tip>
  </Step>
</Steps>

## Invite your first user

<Steps>
  <Step title="Sign in to Signon as a superadmin">
    Open Signon in your browser and sign in with a superadmin account. In the GOV.UK Docker environment, a superadmin user is seeded automatically.
  </Step>

  <Step title="Invite the user">
    1. Click **Users** in the navigation.
    2. Click **Invite user**.
    3. Enter the user's name and email address.
    4. Click **Send invitation email**.

    Signon sends an invitation email to the address you provided. The user follows the link to set their password and activate their account.
  </Step>

  <Step title="Grant the user access to your application">
    After the user's account is active:

    1. Find the user in **Users** and click **Edit**.
    2. Click **Manage permissions** next to your application.
    3. Enable the **signin** permission.
    4. Click **Update permissions**.

    The user can now sign in to your application through Signon.

    <Note>
    Without the `signin` permission, users are redirected back to Signon with an access denied error even if they have a valid session. Always grant `signin` explicitly.
    </Note>
  </Step>
</Steps>

## Verify the integration

Open your application in a browser. You should be redirected to Signon's sign-in page. After signing in, Signon redirects you back to your application's callback URL (`/auth/gds/callback`) and your application creates a session.

If the redirect does not happen, check that:

- `OAUTH_ID` and `OAUTH_SECRET` match the values printed by the rake task.
- The `redirect_uri` registered in Signon exactly matches the callback URL your application uses.
- `GDS_SSO_STRATEGY=real` is set if you are running in development mode.
